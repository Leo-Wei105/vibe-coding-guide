## 🎯 Trae 是什么？

### 官方介绍

**Trae** 是字节跳动推出的AI智能编程工具，专为中文用户优化。它集成了强大的AI能力，让编程变得前所未有的简单。

### Trae的核心特点

| 特点 | 说明 |
|-----|------|
| 🎯 中文优化 | 专为中文用户设计，理解中文需求更准确 |
| 🚀 完全免费 | 目前所有功能完全免费使用 |
| 🤖 多模型支持 | 集成豆包、DeepSeek、Claude、GPT-4o等多种AI模型 |
| 🎨 简单易用 | 界面简洁，零基础也能快速上手 |
| 💡 智能补全 | 强大的代码自动补全功能 |
| 🔧 自动修bug | 自动发现并修复代码错误 |
| 🌐 多语言支持 | 支持Python、JavaScript、Java、C++等多种语言 |

#### 场景2：解释代码

```
（选中一段代码后）

你：这段代码是什么意思？

AI：这段代码实现了一个防抖函数，功能如下：

1. 防止函数在短时间内被多次调用
2. 常用于搜索输入、滚动事件等场景
3. 可以提高性能，减少不必要的计算

具体流程：
- 用户停止输入后等待500ms
- 如果期间没有新的输入，才执行函数
- 如果有新输入，重新计时

这样可以在用户完成输入后才触发操作！
```

```
- 错误信息表示访问了undefined对象的x属性
- 可能原因：
  1. 变量未定义就使用
  2. 返回值是undefined
  3. 异步操作时机问题

建议修复：
1. 检查变量是否已定义
2. 添加默认值处理
3. 使用可选链操作符?.

需要我帮你修复吗？(y/n)
```

## 🚀 高级功能详解

### 1. 两种核心模式：Chat vs Builder ⭐ 重要

Trae 有两种主要的工作模式，理解它们的区别非常重要！

#### Chat 模式（聊天模式）

**快捷键**: `Cmd/Ctrl + U`

**特点**：
- 💬 像聊天一样和AI对话
- 📝 适合问答、解释、小修改
- 🔄 可以查看历史对话
- ⏪ 支持回退到之前的版本

```
┌─────────────────────────────────────────┐
│  💬 Chat 模式                          │
├─────────────────────────────────────────┤
│                                         │
│  你：这个函数是什么意思？               │
│                                         │
│  AI：这个函数的作用是...                │
│                                         │
│  你：帮我加个注释                       │
│                                         │
│  AI：好的，已添加注释...                │
│                                         │
│  [发送] [回退] [历史]                   │
└─────────────────────────────────────────┘
```

**最佳使用场景**：
- ✅ 询问代码问题
- ✅ 解释代码功能
- ✅ 小范围代码修改
- ✅ 学习和探索

#### Builder 模式（构建模式）

**快捷键**: `Cmd/Ctrl + U` → 点击左上角切换到 "Builder"

**特点**：
- 🏗️ 自动创建完整项目
- 📁 可以操作多个文件
- 🖥️ 可以执行终端命令
- 🔍 主动分析项目结构

```

│  📝 创建 style.css ✓                    │
│  📝 创建 app.js ✓                       │
│  🖥️ 运行 npm install ✓                  │
│                                         │
│  项目已创建完成！                        │
│                                         │
│  [接受全部] [逐个查看] [拒绝]           │
└─────────────────────────────────────────┘
```

**最佳使用场景**：
- ✅ 从头创建项目
- ✅ 多文件修改
- ✅ 添加新功能
- ✅ 重构代码

#### 模式对比

| 方面 | Chat 模式 | Builder 模式 |
|------|-----------|--------------|
| **用途** | 问答、小修改 | 创建、大改动 |
| **文件操作** | 单文件为主 | 多文件自动 |
| **命令执行** | 需要确认 | 自动执行 |
| **适合** | 学习、调试 | 开发、构建 |
| **速度** | 即时响应 | 需要等待 |

### 2. SOLO 模式（自主编程）⭐ 高级功能

**SOLO 模式** 是 Trae 2.0 的核心功能，让 AI 可以**完全自主**完成复杂的编程任务。

#### 什么是 SOLO 模式？

想象一下：
- 👨‍🍳 **普通模式**：你是厨师，AI 是助手，你说一步他做一步
- 🤖 **SOLO 模式**：你是老板，AI 是厨师，你说想吃什么，AI 自己搞定

#### SOLO 模式的特点

| 特点 | 说明 |
|------|------|
| 🧠 **完全自主** | AI 自己规划、执行、检查 |
| 📊 **进度可视** | 可以看到 AI 的思考过程 |
| 🔄 **自动迭代** | 遇到问题自动修复 |
| ⏸️ **可以干预** | 随时暂停或调整方向 |

#### 如何使用 SOLO 模式

```
1. 打开 Builder 模式
2. 在输入框右侧找到 "SOLO" 按钮
3. 点击切换到 SOLO 模式
4. 描述你的需求
5. 让 AI 自主完成
```

#### SOLO vs Builder 的区别

| 方面 | Builder | SOLO |
|------|---------|------|
| **交互方式** | 对话式 | 任务式 |
| **AI 自主性** | 需要确认 | 完全自主 |
| **适合任务** | 具体功能 | 完整项目 |
| **用户参与** | 较多 | 较少 |

### 3. MCP 服务器扩展 ⭐ 高级功能

**MCP (Model Context Protocol)** 让 Trae 可以连接外部服务，获得更多能力。

#### 什么是 MCP？

MCP 就像给 Trae 装上"外挂模块"：
- 📚 连接文档系统 → 读取 Notion、语雀
- 🗄️ 连接数据库 → 查询和分析数据
- 🔧 连接开发工具 → 操作 Git、Docker

#### 如何添加 MCP 服务器

**步骤**：
```
1. 打开 Trae 设置（齿轮图标）
```

## 🐛 常见问题

### Q1: Trae连接不上AI

```
问题：AI不回复或一直加载

解决方案：
1. 检查网络连接
2. 切换AI模型（豆包/DeepSeek）
3. 关闭后重新打开Trae
4. 检查账号状态
```

### Q2: 生成的代码不能运行

```
问题：AI生成的代码报错

解决方案：
1. 检查是否有遗漏的依赖
2. 使用AI的"调试"功能
3. 清楚地告诉AI具体错误信息
4. 让AI逐步排查问题
```

### Q3: AI理解不了我的需求

```
问题：AI给出的不是我要的

解决方案：
1. 用更清晰的语言描述
2. 分步骤说明需求
3. 提供示例或参考
4. 选择更合适的AI模型
```

### Q4: 项目太大，AI响应慢

```
问题：项目文件多，AI分析很慢

解决方案：
1. 创建 .traignore 文件排除不需要的文件
2. 只让AI处理特定文件
3. 使用"选中代码"来缩小范围
4. 清理不必要的文件
```

### ❌ 问题1：AI一直在转圈不回复

**症状**：发送消息后加载很久没有回复

**解决方法**：
```
按顺序尝试：

2. 点击模型切换（豆包 ↔ DeepSeek）
3. 完全关闭Trae后重新打开
4. 清除对话历史，开始新对话
```

**症状**：复制AI的代码后运行出错

**症状**：AI给的东西和你想要的不一样

**正确的提问方式**：
```prompt
❌ 不好的说法："帮我做个好看的网页"
```

## 📝 本章小结

### 关键要点：

1. **Trae专为中文优化**：中文理解能力最强
2. **完全免费**：所有功能都可以免费使用
3. **简单易用**：零基础也能快速上手
4. **多模型支持**：豆包、DeepSeek、Kimi、Qwen等多种AI模型
5. **适合初学者**：是学习vibe coding的最佳起点

### 三种模式速查表：

| 模式 | 用途 | 快捷键 | 适合场景 |
|------|------|--------|---------|
| **Chat** | 问答、小修改 | `Cmd/Ctrl+U` | 学习、调试 |
| **Builder** | 创建项目 | 切换按钮 | 新功能、多文件 |
| **SOLO** | 完全自主 | Builder内切换 | 复杂任务 |

### 高级功能速查表：

| 功能 | 作用 | 使用方式 |
|------|------|---------|
| **多模态输入** | 图片交流 | 📎 按钮上传 |
| **模型切换** | 选择AI | 点击模型名称 |
| **MCP服务器** | 扩展能力 | 设置中添加 |
| **Skills** | 自定义规范 | 对话创建 |

### 本章学到的魔法咒语：

| 用途 | 咒语要点 |
|------|----------|
| 创建项目 | Builder模式 + 描述内容+风格 |
| 解释代码 | Chat模式 + 选中代码+"请用最简单的话解释" |
| 改进功能 | "请帮我改进，添加[功能]" |
| 修复错误 | 复制错误信息或截图+"请帮我修复" |
| 按设计实现 | 上传设计稿+"请按这个风格实现" |
| 自主任务 | SOLO模式 + 详细需求描述 |

### 三大工具选择建议

| 你的情况 | 推荐工具 |
|---------|---------|
| 完全零基础，想快速开始 | Trae ⭐⭐⭐⭐⭐ |
| 中文环境，中文需求多 | Trae ⭐⭐⭐⭐⭐ |
| 已有一定编程经验 | Cursor ⭐⭐⭐⭐ |
| 专业开发者，需要强大功能 | Claude Code ⭐⭐⭐⭐⭐ |
